Phase 4 9.1.1 메뉴 등록 새 메뉴 항목 등록. 상위메뉴·메뉴명·URL·정렬순서 입력. 최상위 메뉴는 상위메뉴 없이 등록. 관리자 Phase 4 9.1.2 메뉴 수정 메뉴명·URL·정렬순서·사용여부 수정. 관리자 Phase 4 9.1.3 메뉴 삭제 메뉴 삭제(사용여부 N 처리). 하위 메뉴가 있는 경우 삭제 불가. 관리자 Phase 4 9.1.4 메뉴 조회 전체 메뉴 트리 구조 조회. 권한별 접근 가능 메뉴 필터링. 관리자 Phase 4 9.2 코드 관리 시스템 공통 코드(상태값·분류값 등)를 그룹별로 등록·관리. Phase 4 9.2.1 코드 등록 코드그룹 내 새 코드를 등록. 코드값·코드명·정렬순서 입력. 관리자 Phase 4 9.2.2 코드 수정 코드명·정렬순서 수정. 코드값 변경은 시스템 영향 확인 후 처리. 관리자 Phase 4 9.2.3 코드 삭제 코드 삭제(사용여부 N 처리). 실제로 사용 중인 코드는 삭제 불가. 관리자 Phase 4 9.2.4 코드 조회 코드그룹별 전체 코드 목록 조회. 관리자 Phase 4 9.3 권한 관리 역할(Role)을 정의하고 메뉴별 접근 권한을 설정. 역할 예: ADMIN·OPERATOR·INDIVIDUAL·CORPORATE·DEPT_MANAGER. 관리자 Phase 4 9.4 택배사 관리 서비스에서 사용할 택배사 정보를 등록·수정·삭제·조회. 택배사코드·택배사명(한글/영문)·서비스국가·Tracking URL 관리. 관리자 Phase 1 기초코드로 Phase 1 포함 9.5 알림 관리 회원 대상 알림(공지·운송상태 변경 등)을 생성·발송·관리. Phase 4 9.5.1 알림 등록 알림 제목·내용·발송 대상(전체/개인/법인) 설정. 발송 예약 시각 설정 가능. 관리자 Phase 4 9.5.2 알림 삭제 발송 전 미발송 알림 삭제. 발송 완료된 알림은 이력으로 보존. 관리자 Phase 4 9.5.3 알림 발송 등록된 알림을 대상 회원에게 즉시 또는 예약 발송. 발송 채널: 앱 푸시·이메일(추후 확정). 발송 결과(SENT/FAILED) 기록. 관리자 Phase 4 발송채널 추후 확정 9.5.4 알림 조회 알림 발송 이력 조회. 발송상태·발송일시·대상 정보 확인. 관리자 Phase 4 9.6 통계 관리 운송·비용 데이터를 집계하여 경영 현황을 파악하는 통계 기능. Phase 4 9.6.1 운송 통계 기간별·운송수단별·회원별 오더 건수·중량·CBM을 집계하여 운송 현황 통계 제공. 운영자/관리자 Phase 4 9.6.2 비용 통계 기간별 운송비 수입·원가·이익을 집계하여 수익성 분석. 세부 통계 항목은 추후 확정. 운영자/관리자 Phase 4 상세기능 추후 확정 9.7 데이터 관리 시스템 데이터를 안전하게 보관하기 위한 백업 기능. Phase 4 9.7.1 DB 데이터 백업 PostgreSQL 데이터베이스 전체 또는 테이블별 자동 백업. 백업 주기: 일별·주별 설정 가능. 백업 파일은 MinIO에 저장. 관리자 Phase 4 10 고객지원 회원이 서비스 이용 중 발생하는 문의·공지 정보를 제공하는 기능. Phase 4 10.1 QnA 회원이 1:1 문의를 등록하고 관리자가 답변하는 기능. 문의 등록 후 처리상태(접수·처리중·완료) 확인 가능. 회원/관리자 Phase 4 10.2 FAQ 자주 묻는 질문과 답변을 카테고리별로 관리. 관리자가 FAQ를 등록·수정·삭제하며 회원은 조회만 가능. 회원/관리자 Phase 4 10.3 공지사항 서비스 변경사항·이벤트 등 공지를 등록·관리. 관리자가 등록하고 전체 회원에게 공개. 중요 공지는 상단 고정 가능. 회원/관리자 Phase 4 【범례】 대분류(L1) 중분류(L2) 소분류(L3) 세부기능(L4) Phase 1 Phase 2 Phase 3 Phase 4
