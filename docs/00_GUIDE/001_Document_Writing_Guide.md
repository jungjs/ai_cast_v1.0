## 🗂️ 폴더 및 파일 명명 규칙

### 1. 폴더 명명 규칙

#### 형식
```
[두자리숫자]_[폴더명]

예:
✅ 000_GUIDE
✅ 08_Self_Audit
✅ 090_TEMPLATES
✅ 093_Reference
```

#### 규칙
- **번호**: 두자리 숫자 (00-99)
- **폴더명**: 영문 (한글 불가)
- **구분자**: 언더스코어(_)
- **대소문자**: 대문자 권장

#### 번호 범위
```
000-010: 프로젝트 기본 가이드 & 방법론
020-030: 설계 & 문서
040-050: 테스트 & 품질
060-070: 배포 & 운영
080-090: 관리 & 추적 (SAR, 템플릿)
093-099: 참조 & 외부 자료
```

---

### 2. 파일 명명 규칙

#### 형식
```
[세자리숫자]_[파일명].md

예:
✅ 000_README.md
✅ 001_Document_Writing_Guide.md
✅ 201_SAR_RULE.md
✅ 202_Check_List_Procedure.md
```

#### 규칙
- **번호**: 세자리 숫자 (000-999)
- **파일명**: 영문 PascalCase 또는 Snake_Case
  - ✅ `Document_Writing_Guide` (권장)
  - ✅ `DocumentWritingGuide` (가능)
  - ❌ `document-writing-guide` (kebab-case 금지)
  - ❌ `documentWritingGuide` (camelCase 금지)
- **확장자**: .md (마크다운)

#### 특수 파일
```
각 폴더의 인덱스: 000_README.md
예: 000_README.md
    000_README.md
```

## 📝 메타데이터 표준

### 1. 필수 메타데이터

모든 문서 최상단에 다음 메타데이터를 포함합니다:

```markdown
# [문서 제목]

> **문서 ID**: 0XX  
> **분류**: [분류명]  
> **목적**: [문서의 목적 (한 줄)]  
> **대상**: [대상 독자]  
> **작성일**: YYYY-MM-DD  
> **최종 수정**: YYYY-MM-DD  
> **작성자**: [작성자 또는 팀명]  
> **버전**: v1.0  

---
```

#### 각 항목 설명

| 항목 | 형식 | 예시 |
|------|------|------|
| **문서 ID** | `0XX` (세자리 숫자) | `001`, `102`, `201` |
| **분류** | 이모지 + 분류명 | `📖 문서 작성`, `🎯 방법론` |
| **목적** | 한 줄 요약 (50자 이내) | `이 프로젝트의 모든 문서 작성 표준 정의` |
| **대상** | 누가 읽을 것인가 | `모든 팀원`, `개발자만` |
| **작성일** | `YYYY-MM-DD` | `2026-04-08` |
| **최종 수정** | 처음 = 작성일, 수정 후 업데이트 | `2026-04-08` |
| **작성자** | 이름 또는 역할 | `Claude Code (AI)`, `Team Lead` |
| **버전** | `vX.X` | `v1.0`, `v1.1`, `v2.0` |

### 2. 버전 번호 규칙

```
v1.0  → 초안 (Draft)
v1.1  → 소규모 수정 (Minor fix)
v1.2  → 중간 수정 (Medium update)
v2.0  → 주요 개정 (Major revision)
```

### 3. 버전 이력 (선택사항)

문서에 여러 버전이 있으면 포함:

```markdown
## 📝 버전 이력

| 버전 | 날짜 | 작성자 | 변경 사항 |
|------|------|--------|---------|
| v1.1 | 2026-04-09 | Claude Code | 섹션 3 추가 예시 |
| v1.0 | 2026-04-08 | Claude Code | 초안 작성 |
```

## 🏗️ 문서 구조

### 1. 기본 구조

```markdown
# [문서 제목]

> [메타데이터]
---

[선택사항: 목차]

## 섹션 1
## 섹션 2
## 섹션 3

---

[선택사항: 버전 이력]
---

[네비게이션 하단]
```

### 2. 제목 계층 구조

```markdown
# H1: 문서 제목 (필수, 한 번만)
  → "# 001_Document_Writing_Guide"

## H2: 주요 섹션 (이모지 + 섹션명)
  → "## 📋 목차"
  → "## 🗂️ 폴더 명명 규칙"

### H3: 하위 섹션
  → "### 1. 폴더 명명 규칙"

#### H4: 상세 섹션
  → "#### 형식"
```

#### 규칙
- H1: 한 번만 사용, 문서 제목
- H2부터: 이모지 + 섹션명
- 최대 4단계까지 (H1-H4)

### 3. 목차 (선택사항)

문서가 길면 (> 3개 섹션) 목차 포함:

```markdown
## 📋 목차

1. [섹션 1](#섹션-1)
2. [섹션 2](#섹션-2)
3. [섹션 3](#섹션-3)

---
```

- 문서 끝에 `---` (구분선)

#### 권장 사항

## 📏 파일 크기 관리

### 1. 목표 라인 수 (권장)

| 범위 | 평가 | 조치 |
| --- | --- | --- |
| **500-800줄** | ✅ 최적 | 그대로 유지 |
| **800-1000줄** | ✅ 적절 | 가능하면 유지 |
| **1000-1500줄** | ⚠️ 주의 | 분할 검토 필수 |
| **> 1500줄** | ❌ 과다 | 반드시 분할 |

### 2. 파일 크기 기준 (바이트)

```
최적: < 100KB
적절: 100-500KB
주의: 500KB-1MB (분할 검토)
과다: > 1MB (분할 필수)
```

### 3. 초과 시 분할 전략

**파일이 1000줄을 초과하면 다음과 같이 분할:**

```
[파일명]_Overview.md (800줄 이내)
  ├─ 핵심 내용
  ├─ 기본 사용법
  ├─ 필수 규칙만
  └─ 다른 파일 링크

[파일명]_Detail.md (800-1000줄)
  ├─ 심화 내용
  ├─ 복잡한 사례
  ├─ 엣지 케이스
  └─ 트러블슈팅

[파일명]_Examples.md (선택, 600-800줄)
  ├─ 실제 예시 5-10개
  ├─ 좋은 예 vs 나쁜 예
  └─ 템플릿

000_[분류]_README.md (인덱스)
  └─ 모든 관련 파일 목록
```

### 4. 분할 예시

#### 예: SAR 규칙이 1500줄로 확장된 경우

**현재:**
```
201_SAR_RULE.md (450줄) ✅
```

**분할 후:**
```
201_SAR_Overview.md (800줄)
  ├─ 파일명 규칙
  ├─ 8가지 분류 요약
  ├─ 심각도 요약
  └─ [상세 파일] 링크

202_SAR_Detail.md (700줄)
  ├─ 필수 섹션 상세
  ├─ 작성 방법 심화
  ├─ 분류별 선택 기준
  └─ 트러블슈팅

203_SAR_Examples.md (600줄)
  ├─ 분류별 예시 5-8개
  ├─ 좋은 SAR / 나쁜 SAR
  └─ 실행 템플릿

200_SAR_README.md (인덱스)
  ├─ 201: Overview (기본)
  ├─ 202: Detail (심화)
  └─ 203: Examples (실습)
```

### 5. 분할 판단 체크리스트

```
파일을 분할해야 하는가?

□ 파일이 1000줄 이상인가?
  YES → 분할 검토 시작

□ 여러 주제가 섞여있는가?
  YES → 주제별로 분할

□ 독자가 모두를 읽을 필요가 있는가?
  NO → 개요/상세로 분할

□ 특정 부분만 자주 업데이트되는가?
  YES → 그 부분을 별도 파일로

□ 목차가 너무 길어졌는가? (> 20개)
  YES → 분할 고려
```

## ✅ 체크리스트

새 문서 작성 시 확인:

### 파일 생성 전
```
□ 번호 결정 (000-999, 분류에 맞는 범위)
□ 폴더 결정 (000_GUIDE 등)
□ 파일명 결정 (세자리번호_파일명.md)
□ 폴더의 000_*_README.md 확인
```

### 파일 작성
```
□ 메타데이터 포함 (필수 7개 항목)
  □ 문서 ID
  □ 분류
  □ 목적
  □ 대상
  □ 작성일
  □ 최종 수정
  □ 작성자
  □ 버전

□ 제목 구조 (H1 한 번만)
□ 이모지 사용 (H2+)
□ 상대 경로 링크

□ 목차 (선택, 긴 문서는 필수)
□ 네비게이션 (상단 & 하단)
```

