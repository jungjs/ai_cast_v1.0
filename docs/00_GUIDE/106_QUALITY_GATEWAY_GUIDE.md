# [106] 제니스 물류 플랫폼 2단계 품질 게이트웨이 가이드 (v1.0)

> **문서번호**: 106 
> **작성자**: Antigravity (CEO Agent)  
> **상태**: 공식 채택 (Official Standard)

본 문서는 ZENITH_LMS 프로젝트의 품질 무결성을 확보하기 위한 **[2단계 품질 게이트웨이(Two-Phase Quality Gateway)]** 방법론에 대해 정의합니다. 모든 코드 배포 및 검증 절차는 본 가이드를 최우선적으로 준수해야 합니다.

---

## 🏛️ 제1원칙: 품질은 논리로부터 시작된다
품질 검증은 '화면이 보이는가'가 아닌 '논리가 정밀한가'에서 출발합니다. 이를 위해 검증 프로세스를 이원화합니다.

### Phase 1: 논리적 실드 (Logic Shield - Automated)
논리적 처리와 로직의 무결성을 코드로 증명하는 단계입니다.
- **도구**: Vitest (Unit / Integration Test)
- **대상**: 
  - 비즈니스 알고리즘 (요율 계산, 환율 변동 등)
  - 데이터 스키마 밸리데이션 (Zod Schema)
  - 권한 처리 로직 (RBAC Rules)
- **성공 요건**: 모든 테스트 케이스 100% 통과 (Coverage 80%+ 권장)

### Phase 2: 운영의 혈맥 (Operational Pulse - Scenario-based)
증명된 논리들이 실제 운영 화면(UI)과 결합되어 유기적으로 동작함을 입증하는 단계입니다.
- **도구**: Playwright (E2E), Manual Browser Testing
- **대상**:
  - 실제 비즈니스 시나리오 (예: 오더 생성 -> 배송 완료)
  - UI/UX 반응성 및 다국어 렌더링
  - DB 통합 데이터 정합성 (UI 조작 $\to$ DB 저장)
- **성공 요건**: 사전에 정의된 UAT 시나리오의 모든 단계가 실제 화면에서 오차 없이 수행됨

---

## 🏛️ 제니스 통합 테스트 명세서(ZEN Integrated Test Spec) 작성 표준

모든 테스트 시나리오는 단순한 행위 기술을 넘어 다음의 **[4계층 구조]**를 포함하여 수직적 무결성을 증명해야 합니다.

1. **도메인 및 ID (Domain ID)**: R, O, C, S 등 비즈니스 도메인과 고유 번호 식별.
2. **논리 실드 (Phase 1: Logic Shield)**: 로직 무결성을 검증할 `Vitest` 테스트 파일 및 핵심 함수 명시.
3. **운영 혈맥 (Phase 2: Operational Pulse)**: 실제 UI 조작 경로 및 기대 반응 명시.
4. **통합 매트릭스 (Integrated Matrix)**: Phase 1(로직)과 Phase 2(화면)의 성공 여부를 한눈에 기록.

---

## 🛠️ 검증 운영 표준 (SOP v2.0)

1. **통합 명세서 작성**: 검증 전 `docs/08_Self_Audit/UAT/` 하위에 통합 시나리오를 작성한다.
2. **Phase 1 (Logic) 수행**: 엔진의 논리적 버그를 선제적으로 제거하고 코드 증거를 남긴다.
3. **Phase 2 (Screen) 수행**: 입증된 엔진 위에서 실제 화면 시나리오를 전수 검증한다.
4. **중앙 집중식 결과 기록**: 모든 검증 결과와 특기 사항은 해당 시나리오 파일의 [매트릭스] 섹션에 즉시 기록하여 보존한다.

---

## 🏛️ 제3실천: 지속적 품질 자산화 (CQA - Mandatory)

제니스의 품질은 축적되어야 합니다. 모든 신규 개발 및 수정 사항은 다음의 **[회귀 프로토콜]**을 통과해야 합니다.

1. **상시 증명**: 모든 작업 완료 보고 전, 표준 명령어(`npm run test:regression`)를 수행하여 기존 기능의 파괴 여부를 자가 진단한다.
2. **증거 기반 승인**: 에이전트는 작업 완료 보고 시 회귀 테스트 성공 결과를 증거로 첨부하여 품질 무결성을 시각적으로 입증한다.
3. **자산화**: 신규 기능 개발 시, 해당 기능을 위한 Phase 1/Phase 2 테스트를 기존 수트에 누적(Cumulative)하여 미래의 품질 관문으로 삼는다.

---

## 📈 품질 확보 기대 효과
- **회귀 테스트 보장**: 코드 수정 시 로직 결함 여부를 수초 내에 자동 파악 가능.
- **사용자 신뢰도 극대화**: 실제 운영 화면에서의 유기적 동작을 직접 증명하여 실무 투입 리스크 최소화.
